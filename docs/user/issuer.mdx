# Issuers

An issuer is a person/entity that issues NFTs or collections. By issuance, we understand the process of creating and listing in a primary market. In other words, it is the "content creator". The issuer can be the client itself, which decides to issue its own NFTs or any content creator that is associated with the client, e.g. if the client is a museum, the museum can issue its own NFTs or it can create issuer accounts for each of its collaborating artists and let them issue NFTs.

- Create issuers

<Highlight color="#079CEE">POST</Highlight> <code>/api/v1/issuers</code>

#### Request Body Fields

The fields below should be formatted as `json` elements and put in the request body as shown in the example below.

```json
    {
        "basicInfo": {
        "name": "Salvador Dali",
        "countryCode": "US",
        "taxIdentificationNumber": "586924667"
    }
    }

    response:

    {
        "id": "4def6390-fcd7-4ea7-85fb-26e30988b260",
    "basicInfo": {
        "name": "Salvador Dali",
        "countryCode": "US",
        "taxIdentificationNumber": "586924667"
    },
    "dltCredentials": {
        "publicKey": "N/A",
        "dltAddress": "0xDC6725D6714cED8B6915763A3C3A79BC91271BCd"
    },
    "isCustodial": true,
    "status": "ACTIVE",
    "_links": {
        "self": {
            "href": "https://api.pre.dats.io.builders:443/api/v1/issuers/4def6390-fcd7-4ea7-85fb-26e30988b260",
            "templated": false
        }
    },
    "_embedded": {}
}
```

| Field      | Required | Description                 |
| :--------- | :------- | :-------------------------- |
| `username` | Yes      | The usernames input by user |
